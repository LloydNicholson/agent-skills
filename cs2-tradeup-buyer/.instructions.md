# CS2 Trade-Up Buyer — CLI Workflow

This skill finds, scouts, and executes profitable CS2 trade-up opportunities using the installed `cli-anything-cs2marketbot` CLI.

## Important

- Use the CLI only. Do not call the web API directly.
- Prefer JSON output for every step where you need to parse data.
- Treat the running `CS2MarketBot.Web` server as a hard dependency.
- If the CLI is not installed or the server is unreachable, fail clearly and tell the user what to start.

## Configuration

Default server URL is stored in the CLI session config, but can be overridden with `--base-url`.

To set the preferred server permanently:

```bash
cli-anything-cs2marketbot session config --set-url https://cs2marketbot.lloydindevelopment.work
```

## Workflow

### 1) Find candidate trade-ups

Use the CLI to list opportunities and choose the best match for the user’s budget and filters.

```bash
cli-anything-cs2marketbot --json opportunities list   --max-cost <budget>   --unique-by-output   --sort profit   --page-size 5
```

If the user asked for StatTrak, type filtering, or a minimum profit threshold, include the matching CLI flags.

### 2) Scout the selected opportunity

Start a scout-only run:

```bash
cli-anything-cs2marketbot --json execute scout <opportunity-id>
```

Poll the job until it finishes:

```bash
cli-anything-cs2marketbot --json execute status <job-id>
```

If the result is partial or short, explain the shortfall and ask the user whether to try another opportunity.

### 3) Ask before purchase

Only after a successful scout and a clear summary, ask the user for confirmation before purchasing.

### 4) Execute the purchase

If approved, run:

```bash
cli-anything-cs2marketbot --json execute start <opportunity-id>
```

Then poll status again until completion.

### 5) Keep the user informed

Report:
- opportunity ID
- output skin
- total cost
- profit percent
- profit chance
- job ID
- final purchase result

## Error handling

- If the CLI returns a non-zero exit code, capture stderr and surface the error.
- Retry transient HTTP failures once or twice if the error looks network-related.
- If there are no opportunities, ask the user to widen budget or relax filters.
- If the server is down, tell the user to start `CS2MarketBot.Web` and try again.

## Do not

- Do not use the old REST/MCP workflow directly.
- Do not invent opportunity data.
- Do not purchase without confirmation.

## Good phrasing

Use concise status updates like:
- "Found 4 opportunities under R20."
- "Scouting opportunity #27563035."
- "Scout succeeded; 10/10 inputs were found."
- "Ready to buy — proceed?"
